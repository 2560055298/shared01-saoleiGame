HGE::Gfx_StartBatch

启动用户图形元（graphic primitives）批处理的渲染。 

hgeVertex *Gfx_StartBatch(
  int prim_type,
  HTEXTURE tex,
  int blend,
  int *max_prim
);

参数

prim_type
要渲染的图形元（graphic primitives）的类型:
HGEPRIM_LINES - 直线
HGEPRIM_TRIPLES - Triples
HGEPRIM_QUADS - Quads
tex
要使用的纹理句柄或者0.
blend
混合模式。
max_prim
指向int的指针，用来接收一次批处理中允许的最大图形（primitive）数。

返回值

如果执行成功，则返回指向存有顶点坐标的顶点缓冲的指针并将允许的最大图形数存入max_prim，否则返回0。

备注

Gfx_StartBatch是通向 HGE 和 DirectX核心的后门，用于专业使用。 它提供了直接访问硬件顶点缓冲并允许高速渲染大型图形元(primitive)数组。主要用于复杂运算，纹理效果和复杂的预计算对象的渲染。

Gfx_StartBatch函数必须在Gfx_BeginScene和Gfx_EndScene函数之间调用。 要结束一个批处理可以使用Gfx_FinishBatch函数。 不允许在Gfx_StartBatch/Gfx_FinishBatch调用对之间使用其他渲染函数。

顶点缓冲的大小是常量不可以改变。它可以保存1000个quad/1333个triple/2000个line。 向缓冲中写入时，每个Line需要2个顶点，每个triple需要3个，每个quad需要四个。 实际写入缓冲的图形元数量（primitive）应该传给Gfx_FinishBatch。

Gfx_StartBatch会冲掉所有之前积累的图形元。

需要

头文件: hge.h
库文件: hge.lib

另见

Gfx_FinishBatch, Gfx_BeginScene, Gfx_EndScene
