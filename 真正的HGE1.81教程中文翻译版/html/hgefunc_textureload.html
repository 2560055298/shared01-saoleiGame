HGE::Texture_Load

从内存，资源包或磁盘文件中加载一个纹理。 

HTEXTURE Texture_Load(
  const char *filename,
  DWORD size = 0,
  bool bMipmap = false
);

参数

filename
纹理文件名。
size
如果此参数不为零，它代表拥有已知格式纹理的内存块大小，参数filename被看做是这块内存的指针。
bMipmap
如果为true,将自动创建一套MIP映射层级用于渲染。

返回值

如果执行成功返回纹理句柄，否则返回0.

备注

支持BMP, DDS, DIB, JPG, PNG 和 TGA图形文件格式。 DDS格式支持DXT1-DXT5纹理压缩技术，DXT1-DXT5有高效低需求的特点，可以再使用更少的显存的情况下有高效的表现。

纹理维数必须是2的幂数，如果此条件不满足，纹理维数将被自动转换为最接近的2的幂数值的维数，大于纹理的实际维数。

如果filename代表相对路径，纹理文件首先在所有资源包范围内检索并通过Resource_AttachPack载入。 然后再从程序磁盘文件夹检索。 如果filename代表绝对磁盘路径，纹理文件将直接从磁盘载入。

如果使用了MIP映射请求，将生成一系列对应的小号版本的纹理来用于渲染。 这将会产生更高效高质量的精灵（sprits）渲染，而使用他们的原尺寸纹理效率和质量都将会降低。 使用此选项的纹理需要额外的1/3的显存。

如果函数调用失败，可以使用System_GetErrorMessage函数来获得错误信息。 另外如果设置了HGE_LOGFILE系统标记，错误信息将被写入日志。

当纹理不再被使用时，必须使用Texture_Free函数将其删除。

需要

头文件: hge.h
库文件: hge.lib

另见
