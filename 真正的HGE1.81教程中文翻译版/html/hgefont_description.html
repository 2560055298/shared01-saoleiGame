字体描述文件格式

要创建并使用hgeFont对象，你需要一组描述你的字体的文件。

字体描述文件

此文件必须传给hgeFont constructor。 它仅仅是包含了所有字体负号说明的文本文件。

第一行应该包含字体描述文件的标识： 

[HGEFONT]

紧随其后是字体位图文件名称（路径是相对于描述文件本身的）： 

Bitmap=font_bitmap.png

接下来是字体中每个字符的描述： 

Char=" ",1,1,3,30,-1,4
Char="!",5,1,7,30,1,0
Char=""",13,1,8,30,3,2
...
Char=FE,445,187,17,30,0,0
Char=FF,463,187,16,30,-2,-1

每行中的第一个值代表要描述的字符。可以是双引号括起来的字符本身，或者是它们的十六进制码。

然后是4个数字，定义字符在字体位图上的位置和大小：x坐标，y坐标，字符的宽度和高度。

最后两个数字定义了成行文本渲染时字符的水平位置偏移量。 第一个是字符前方偏移量，第二个是字符后方偏移量。 偏移量可以是负数。这使得斜体，手写体和艺术体字成为可能。

一个完整的字体描述文件可能会是这样的： 

[HGEFONT]

Bitmap=font_bitmap.png

Char=" ",1,1,3,30,-1,4
Char="!",5,1,7,30,1,0
Char=""",13,1,8,30,3,2
...
Char=FE,445,187,17,30,0,0
Char=FF,463,187,16,30,-2,-1

字体位图文件

此文件必须存有字体描述文件中所描述的所有字符的图像。字体位图文件的维数必须是2的幂次方。
